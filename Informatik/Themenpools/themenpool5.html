Software Development Kit - SDK

  1. Entwickler benötigen passendes SDK für das gewählte Betriebssystem
  2. SDKs: Umfassen:
    1. Dokumentation
    2. APIs - Application Programming Interface
    3. Codebeispiele - examples
    4. Bibliotheken und Prozesse sowie Anleitungen
  3. Insbesondere umfassen SDKs:
    1. Bibliotheken oder APIs: vordefinierte Codeteile mit denen Entwickler allgemeine Programmieraufgaben auf der Plattform ausführen können
    2. Integrierte Entwicklungsumgebung ( Integrated Development Environment - IDE): Ein visueller Editor, mit dem Entwickler grafische Elemente wie Textfelder und Schaltflächen entfwerfen und gestalten können
    3. Tools, mit denen Entwickler Aufgaben wie das Debuggen
      Erstellen
      Ausführen und
      Testen
      ihrer Anwendungen ausführen können
  4. SDKs erweitern Apps um zusätzliche Funktionen und Push-Benachrichtigungen, Anzeigen usw.
    1. Beispiel: Bilder oder Text aus einer App direkt auf Facebook teilen - Android SDK von Facebook
    2. Beschleunigt die Bereitstellung, da Entwickler den Code nicht von Grund auf neu schreiben muss

Entwicklung Mobiler Applikationen

iOS Google Android
iOS Apps erstellt man in der IDE Xcode:
• Entwicklungsumgebung speziell für Swift und Objective-C
• unterstützt aber ebenfalls C und C++
• Software Development Kit (SDK) direkt in Xcode → nur Bibliotheken davon verfügbar (z.B. HomeKit) 		Android App benötigen Android Software Development Kit (SDK):
• wird mit der Entwicklungsumgebung „Android Studio“ ausgeliefert
• meistgenutzte Programmiersprache für Android Apps ist Java.
• auch C++ oder Ruby werden häufig genutzt
iOS-Geräte laufen oftmals auf der aktuellsten Version aufgrund des geschlossenen Quellcodes (keine Mods verfügbar) Android befindet sich auf einer Vielzahl von Geräten → unterschiedliche bzw. modifizierte Versionen von Android im Umlauf
App-Einreichungsprozess bei Apple etwas strenger Das Erstellen einer Android App bringt mehr Aufwand mit sich, als das Entwickeln einer iOS App → Geräte- und Versionen Vielfalt
Veröffentlichung im Apple AppStore kann bis zu einer Woche dauern:
• Apple Review Team überprüft die Qualität und Funktionalität der eingereichten App
• Bei Fehlern müssen zuerst Verbesserungen erfolgen, bevor die App veröffentlicht werden darf 		Veröffentlichen im Google PlayStore relativ rasch und unkompliziert:
• In der Google Developer Konsole lädt man die erstellte App hoch
• Innerhalb von 2–3 Stunden ist die App veröffentlicht.
Apple verlangt für jedes Entwicklerkonto jährlich ca. 99$ US-Dollar Im Google PlayStore ist eine einmalige Gebühr von 25$ US-Dollar zu bezahlen
