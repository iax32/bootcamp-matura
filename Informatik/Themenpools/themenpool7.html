A - Hacker und ihre Methoden

Hacker und ihre Methoden

White Hat Hacker

Arbeiten innerhalb des Gesetzes und der Hackerethik

Arbeitsbereich: Ausführung von Penetrationstest und Aufdeckung von Sicherheitslücken im Unternehmen ( werden von disen engagiert) - Gut

Grey Hat Hacker

Verstoßen möglicherweise gegen das Gesetzt und haben eine restriktive Auslegung der Hackerethik

Arbeitsbereich: Veröffentlichen von Sicherheitslücken, zwingt die verantwortlichen dazu diese zu beheben - weder gut noch böse

Black Hat Hacker

handeln mit krimineller Energie im auftrag von Regierungen oder Organisationen - "Cyberkrieg" - absichtliches beschädigen von Zielsystemen oder Datendiebstahl

Arbeitsbereich: Widerrechtlichen Manipulation von Software, Teil der illegalen WareSzene, Abgrenzung zur legalen Cracker Szene begeisterter Programmierer

Ethische Grundsätze sind:

Zugang zu Computern soll frei sein / Alle Informationen müssen frei sein / Misstraute Autoritäten fördert Dezentralisierung / Beurteile einen Hacker nach dem, was er macht / nicht nach üblichen Kriterien (Aussehen, Alter usw.) / Computer können Kunst und Schönheit schaffen / Computer können Leben zum Besseren verändern / Stöbere nicht in den Daten anderer Leute / Öffentliche Daten nützen, private Daten schützen

Methoden

Virus/Würmer

Virus erstellt mehrere kopien von sich auf dem Gerät - Ein Wurm ebenfalls aber automatisch durch eine Sicherheitslücke im Netzwerk

Social Engineering

Gesellschaftliche Kontakte bringen begehrten Information

Trojanisches Pferd

Als nütztliche Anwendung getarnt. Erfüllt im Hitergrund, ohne Wissen des Anwenders, eine andere Funktion

Backdoors

Ermöglicht, unter Umgehung der normalen Zugriffsrechte Zugang zum Computer, über Universalpasswort des BIOS, über spezielle Software (Trojaner), die Fernzugriff auf das Computersystem ermöglicht

Rootkits

Objekte und Aktivitäten vor den Augen des Anwenders verbergen - Installation am komprimierten System nach Einbruch in das Computersystem - Führt geheime Prozesse aus und versteckt Daten

DoS - Denial of Service oder DDos Distrubited Denial of Service

Dos -Außerstandsetzung eines Netzwerkdienstes z.B durch Überlastung

Exploit

Nutzt spezifische Privilegien - Ausführung einer DoS-Attacke oder SQL Injection

Vulnerability Scanner

Automatische Analyse von Computersysteme - Hilfsprograme suchen nach Sicherheitslücken in Anwendung, Computer oder Netzwerk - Helfen Anfälligkeiten zu erkennen

Sniffer

Realisiert Datenverkehr eines Netzwerk oder eines angeschlossenen Geräts - Empfang, Aufzeichnung, Darstellung und Auswertung von Daten

Keylogger

Technick zum Aufzeichnen der Tastatureingaben

B - Coding Advanced

Was ist ein Shell

Es ist eine „Mensch-Maschine-Schnittstelle“ - die Software mit dem der Benutzer mit einem Betriebssystem interagiert

Beispiele wären hier CMD & PowerShell von Windows oder das Terminal von Linux oder macOS:

Eine Alternative zur Shell ist die GUI, das „Graphical User Interface“, eine grafische Schnittstelle, die eine visuelle Darstellung einer Interaktion zwischen Mensch und Computer ermöglicht.

Wichtige Linux Shell-Befehle

  1. ls -l (long list)
  2. du (disk usage)
  3. cd / etc (change directory - Verzeichniss /etc)
  4. mkdir verzeichniss (make directory - namens verzeichniss)
  5. touch datei1 (erstellt Datei datei1)
  6. grep (filtert Dateien)
  7. chmod g-r datei (Dateiberechtigung group kann nicht lesen)
  8. chmod o+w datei1 (Dateiberechtigung others können schreiben)
  9. chmod u-x datei1 (Dateiberechtigung user kann nicht ausführen)
  10. df (diskfree - zeigt Speicherplatz Belegung)
  11. ln (hardlink)
    1. Hardlink: Kopie der Datei
  12. ln -s (symlink)
    1. symlink: Verknüpfung der Originaldatei
  13. nano (öffnet nano Texteditor)
