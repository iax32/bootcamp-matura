A - Kommandozeileninterpreter und Guis

Shell

In einer Shell, bzw. einem Command Line Interface (CLI) muss der Benutzer mithilfe von Text mit dem
OS kommunizieren. Hierfür müssen Benutzer die jeweiligen Befehle der Shell kennen, wodurch Shells
deutlich schwerer für neue Nutzer zu bedienen sind. Ein großer Vorteil ist die Exaktheit von Eingaben.
Weiters braucht sie nur wenig Speicher, hat hohe Präzision, ist sehr schnell und braucht nur eine
Tastatur. Dafür kann sie nur wenig modifiziert werden, verwendet keine Grafiken und hat keine
Menüs.

GUI

In einer GUI muss der Benutzer mithilfe von Maus und Tastatur mit dem OS kommunizieren. Der
Benutzer braucht dafür die Befehle des OS nicht kennen und kann einfach mithilfe der Menüs und
Grafiken der GUI herausfinden, wie er den Computer bedienen muss. Dafür benötigt die jedoch viel
Speicherplatz, hat teilweise ungenaue Eingaben, ist langsam und benötigt sowohl eine Maus als
auch eine Tastatur. Tippfehler werden durch die GUI vermieden und der Benutzer kann ohne
Probleme mehrere Dinge am Bildschirm auf einmal sehen.

B - Virtualisierung in der Informatik

Virtualisierung

Generell kann Virtualisierung beschrieben werden als mehrere Software-Systeme auf einer Hardware.
So können Prozessoren virtualisiert werden, Partitionen auf Festplatten gemacht werden, virtuelle
LAN-Netzwerke aufgestellt werden oder gar ganze Computer virtualisiert werden.

Gründe

VMs haben viele Vorteile:
1. Sie können Ausfälle vermeiden (z.B bei Angriffen), da bei einem Absturz immer nur eine
Komponente ausfällt (ein OS, nicht der gesamte Computer).
2. Die Hardware wird deutlich besser ausgelastet, da man sie konsolidieren kann ->
3. führt wiederum zu niedrigeren Kosten und einem niedrigeren Stromverbrauch.

Virtuelle Maschinen

Eine virtuelle Maschine (VM) ist eine Software die Hardware simuliert. So kann in einer VM ein
Betriebssystem installiert werden, welches dann die Hardware der VM als echte Hardware sieht und
glaubt, dass es in einem vollständigen Computer ist. Die Komponenten des Host-Computers stehen
den parallel installieren Betriebssystemen nicht direkt, sondern über die Schnittstellen der VM zur
Verfügung.

Hypervisor

Ein Hypervisor wird manchmal auch als VM-Monitor bezeichnet. Er erstellt und verwaltet virtuelle
Hardware, um zu vermeiden, dass Treiber einer VM an die echte Hardware kommen. Dies hilft auch
dabei, dass sich die Betriebssysteme von mehreren VMs nicht in die Quere kommen. Sie können in
zwei Typen aufgeteilt werden:

- Typ 1 (native): Der Hypervisor läuft direkt auf der Hardware. Dies hat den Vorteil, dass er schneller
ist und weniger Ressourcen braucht. Er benötigt jedoch alle Treiber für die Hardware

- Typ 2 (hosted): Der Hypervisor liegt auf einem vollwertigen OS und verwendet die ihm zur Verfügung stehenden Ressourcen.
