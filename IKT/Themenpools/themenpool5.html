A - Dateisysteme im Vergleich

Windows vs. Unix

Unix ist ein Kommandozeilenbasiertes OS, während Windows ein Menü-basiertes OS ist. Während Windows von Microsoft lizenziert ist, befindet sich Unix unter der General Public License. Unix hat ein Text-basiertes Interface, welches die Benutzung durch neue User erschwert. Windows währenddessen hat ein GUI. Während Unix das stabilere, aber langsamere Multiprocessing unterstützt, verwendet Windows das schnellere, aber instabilere Multithreading. Unix verwendet das Unix File System (UFS), während Windows FAT32 und NTFS verwendet. Im Vergleich zu Windows ist Unix sicherer, da das Ausführen von .exe Dateien deutlich schwerer ist. Auch Viren können nur einen Benutzer infizieren und nicht Root selbst. Windows hat ein integriertes Backup und Recovery System; in Unix muss der Benutzer selbst ein solches System schaffen. Während mittlerweile beide sehr stabil sind, ist Unix immer noch etwas stabiler als Windows. Unix ist immer case-sensitive, während Windows nur optional case-sensitive ist.

Merkmal Unix Windows
Benutzeroberfläche Textbasierend GUI ( Menü-basiert )
Lizensierung Open Source ( General Public License ) Proprietär ( Microsoft-lizensiert ) / Closed Source
Benutzerfreundlichkeit Weniger Einsteigerfreundlich Benutzerfreundlich
Parallelverarbeitung Multiprocessing ( stabil, aber langsamer ) Multithreading ( schneller, aber potenziell instabil )

Dateisystem

Ein Dateisystem ist ein System für den Computer, welches bestimmt wo Dateien logisch platziert sind und wie sie benannt werden. Dateisysteme sind essentiell für die Funktionalität des Geräts, da Dateien ansonsten nicht von anderen Dateien isoliert wären. Man kann von Dateisystemen denken wie von einem Index. Außerdem beinhalten Dateisysteme Metadaten einer Datei und andere Konventionen für z.B. die Benennung von Dateien. Sie können auch Berechtigungen von einzelnen Benutzern regeln.

Ein Dateisystem ist eine zentrale Komponente eines Betriebsystems, die bestimmt:

1. Wie und Wo Dateien logisch auf einem Dateiträger gespeichert werden

2. Wie Dateien bennant werden dürfen ( Namenskonvention )

Funktionen eines Dateisystems:

1. Organisation: Ordnet Dateien strukturiert ( wie ein Index in einem Buch ).

2. Isolation: Trennt Dateien voneinander, damit sie sich nicht gegenseitig beinflussen.

3. Metadaten: Speichert Zusatzinformationen wie:
- Erstellungsdatum
- Größe
- Dateityp
- Zugriffsrechte

4. Berechtigungen: Bestimmt welcher Benutzer was mit einer Datei tun darf ( lesen, schreiben, ausführen )

NTFS

New Technology File System, oder NTFS, ist der Nachfolger der File Allocation Table (FAT) und kann daher Datenträger aus FAT32/16/8, ohne jeden Eintrag neu schreiben zu müssen, konvertieren. Außerdem ist es das standardmäßige Dateisystem von Windows seit Windows NT 3.1 (Juli 1993). Es hat keine Trennung zwischen Index- und Datenbereichen. Dateiblöcke können mithilfe von Bereichsbäumen (B-Trees) aufgefunden werden, was zu einer sehr komplexen Implementierung, jedoch einer schnellen Suche führt. Auch Verzeichnisse sind eine Datei, was zu sehr vielen Metadaten führt. Hat eine Meistertabelle, in der alle Metadaten von allen Dateien gespeichert sind (Master File Table -> MFT).

UFS

Bei UFS werden Dateien in Blöcken im Blockbereich gespeichert. Diese können mithilfe von Index- Nodes im Indexbereich aufgefunden werden. Ein Superblock enthält währenddessen alle wichtigen Informationen des Dateisystems (Bereichsgrößen, Datum, Name, Blockgröße, …).

B - Komponenten von Informatiksystemen

Hardware

Hardware kann allgemein als die greifbaren Komponenten eines Computers bezeichnet werden. Sie kann genauso wie die Software ohne die Hardware nicht funktionieren kann, nicht ohne die Software funktionieren.

Peripheriegeräte

Peripheriegeräte bestehen aus Geräten, die nicht im Computer selbst sind. Diese befinden sich in der Peripherie (daher der Name) des Gerätes. Zu ihnen gehören zum Beispiel Mäuse, Tastaturen oder Monitoren. Auch Festplatten gehören zu ihnen. Sie können in Eingabegeräte, Ausgabegeräte und in Externe Speicher eingeteilt werden.

Zentraleinheit

Die Zentraleinheit hingegen besteht aus Eingabegeräten, eventuell einer GPU, einem Arbeitsspeicher und einer CPU. CPU steht für Central Processing Unit, manchmal auch Main Processor und ist der wichtigste aller Prozessor in einem Informatiksystem. Er besteht aus drei wichtigen Komponenten. Die erste ist die Arithmetische Logikeinheit (ALU), sie kann arithmetische und bitweise Operationen an Integern ausführen. Prozessor Registern, die die Daten aus der ALU kurzfristig speichern, sowie die Kontrolleinheit, die das Fetchen aus dem Speicher, sowie das Dekodieren und Ausführen von Instruktionen kontrolliert. Es gibt ein Gesetz, das besagt, dass sich die Anzahl an Transistoren auf einem Mikrochip-Prozessor alle zwei Jahre verdoppelt, das bis heute wahr war. Jedoch wird es aufgrund von Quantensprüngen vermutlich nicht mehr lange wahr sein können. Dieses ist das sog. Mooresche Gesetz.
Für die Effizienz von Prozessoren gibt es messbare Faktoren; diese sind:

1. Taktfrequenz (in GHz): beschreibt wie häufig pro Sekunde Ladungen verändert werden, umso höher, desto leistungsfähiger sind Prozessor + Busse

2. Datenbreite: Wie viele Operationen zeitgleich ausgeführt werden können, aktuell bei 64-Bit

3. Cache-Speicher (in MHz): Pufferspeicher außerhalb des Prozessors (z.B. RAM) müssen auch schnell und konstant Daten übertragen

4. Zugriffszeit: Wie lange es braucht Datenblöcke abzurufen

PC-Komponenten

Ein PC besteht aus mehreren Komponenten. Zu diesen gehören ein Gehäuse, ein Motherboard, ein Netzteil (>=500W), mindestens ein Prozessor, Wärmeleitpaste und Kühlkörper, RAM, Festplatte, Lüfter (entweder Push oder Pull Luftzirkulation), optional auch weitere Prozessoren wie eine GPU oder CD- Laufwerke etc. Auch Kabelmanagement ist wichtig beim Bau eines PCs.

Software

Software hingegen kümmert sich darum, dass die Hardware mit dem Benutzer kommunizieren kann. Sie kann in unterschiedliche oben bereits genannte Kategorien eingeteilt werden.
