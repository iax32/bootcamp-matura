Themenpool 7: Netzwerktechnologien

A - LAN- und WAN-Systeme

LAN, MAN und WAN

Als Netzwerk kann man eine beliebige Anzahl an Computersystemen bezeichnen, die miteinander verbunden sind und Daten austauschen können. Diese kann man dann in unterschiedliche Kategorien einteilen:

- PAN: Das Personal Area Network,
- LAN: Local Area Network,
- MAN: Metropolitan Area Network,
- WAN: Wide Area Network und das
- GAN: Global Area Network.

PAN

PANs können heute ohne Probleme direkt eingerichtet werden und dienen zum Beispiel zur
Datenübertragung zwischen zwei Geräten, als Uplink oder um IoT oder Peripheriegeräte mit einem
Smartphone oder Computer zu verbinden. Die Verbindung erfolgt meist über USB oder Bluetooth

LAN

Bei LANs können weitere Geräte verbunden werden.Hierzu sind jedoch Strukturen wie:
- Hubs,
- Bridges
- oder Switches
erforderlich. Sie können z.B. drei Geräte in einem Haushalt umfassen oder aber
Tausende in einem Unternehmen. Der am weitesten verbreitete Standard ist Ethernet. Sie dienen
dazu einen schnellen Datenverkehr zu ermöglichen und haben meist Geschwindigkeiten von bis zu
1000 Mbit/s. Oft werden Fileserver oder Drucker mit ihnen verbunden. Rein theoretisch könnten sich
LANs über mehrere Kilometer erstrecken haben jedoch meist nur eine Größe von einem
Gebäudekomplex.

MAN

Ein MAN kann mehrere LANs innerhalb einer Stadt mit sehr schnellen Kabeln verbinden. Meist ist die
Übertragungsgeschwindigkeit gleich schnell wie innerhalb eines LANs. Oft werden so mehrere
Niederlassungen einer Firma verbunden. Werden sie auch überregional verbunden sind sie WANs
und global GANs.

WAN

WANs verbinden mehrere MANs multinational und teilweise über gesamte Kontinente. Die Anzahl der
Benutzer ist hierbei faktisch unbegrenzt. Es kommen andere Protokolle wie IP oder MPLS vor.
Internetprovider nutzen WANs, um Kunden an das Internet anzubinden.

GAN

GANs verbinden mehrere WANs global miteinander über Seekabel oder Satellitenverbindungen. Neben dem World-Wide-Web gibt es auch manche abgeschottete Unternehmensnetzwerke.

P2P - Peer to Peer

P2P oder Peer to Peer ist eine Art Clients zu verbinden. Das Gegenteil von P2P ist eine Client-Server
Verbindung. Bei P2P verbinden sich die Clients direkt miteinander. Die Clients führen sämtliche
Berechnungen aus und speicher alle Daten. Dies kann ein Sicherheitsrisiko sein, hat jedoch den
Vorteil, dass die Kommunikation zwischen zwei Clients schneller abläuft und ist deutlich
kostengünstiger für den Anbieter eines Services. Eines der berühmtesten Beispiele für Peer to Peer
Verbindungen sind Torrents

Switch und Hub

- HUB: ein Hub ist ein Gerät, welches mehrere Geräte miteinander verbinden kann. Er funktioniert passiv.

- Switch: Ein Switch hingegen ist aktiv, es sendet nur dann Pakete, wenn ein Gerät sie anfordert.

B - Servertechnologien und -aufgaben

Serverdefinition

Ein Server kann mit zwei unterschiedlichen Definitionen definiert werden: Die Erste Definition ist die
Definition von der Hardwareseite. Diese beschreibt einen Server als eine in ein Rechennetz
eingebundene physische Maschine. Er kann auch als Host bezeichnet werden und jeder Rechner
mit einer Serversoftware ist ein Server.

Ein Server von der Softwareseite ist ein Programm, das einen speziellen Dients anbietet. Es wird von
Clients über ein Netzwerk in Anspruch genommen und der Dienst hängt von der Art der
Serversoftware ab. Als Kommunikationsgrundlage dient das Server-Client Modell und er verwendet
Kommunikationsprotokolle wie FTP, um Daten auszutauschen.

Windows Server

Windows hat einen sehr komplexen Aufbau, um eine möglichst einfache Bedienung durch den
Benutzer zu ermöglichen. Alle Programme sind mit graphischer Benutzeroberfläche verfügbar.
Während die Administration über die Kommandozeile möglich ist, ist sie nicht notwendig. Der
Benutzer hat die volle Kontrolle über alle Systemressourcen, kann jedoch dadurch auch kritische
Fehler auslösen.

Windows hat zusammengefasst folgende Vorteile:
- Einsteigerfreundliche Bedienung durch GUI,
- Treiber für aktuelle Hardware sehr schnell verfügbar,
- große Anzahl an Drittanwendungen,
- Einfache und automatisierbare Aktualisierung,
- Lösung technischer Probleme via Systemwiederherstellung möglich, - Gesicherter Langzeit Support,
- sowie exklusive Programme, die nur für Windows verfügbar sind.

Jedoch hat es auch folgende Nachteile:
- Hohe Lizenzkosten, die mit jedem Benutzer steigen,
- häufige sicherheitsrelevante Fehler,
- Anfälligkeit für Malware,
- es ist ressourcenaufwändig (insbesondere aufgrund des Pflicht-GUIs),
- es hat ein hohes Nutzer-Fehlerpotenzial,
- Es ist als Multi-User-System ungeeignet und
- die Funktionsweise des proprietären Systems ist nicht vollständig offengelegt

Linux Server

Hat einen sehr simplen Aufbau. Alle Komponenten sind eine Datei (auch Prozesse und Geräte). Bietet
sehr große Freiheiten, was eine Überforderung für unerfahrene Nutzer sein kann. Es gibt auch viele
Tools, die bei der Administration eines Servers helfen. Es muss sowohl die Kommandozeile als auch
das GUI verwendet werden.

Linux hat daher folgende Vorteile:
- Die Nutzung ist kostenfrei,
- Administratoren profitieren von den Freiheiten bei der Verwaltung des Systems,
- Linux unterstützt kooperatives Arbeiten, ohne dass der gewöhnliche Benutzer den Programmkern beschädigen kann,
- es ist selten im Fokus von Cyberkriminalität,
- es gibt selten sicherheitsrelevante Fehler, die dann in der Regel schnell behoben sind,
- es hat geringe Ansprüche an die Hardware, und
-es gibt eine Integrierte Remote-Funktion zur Fernverwaltung.

Auch Linux hat jedoch seine Nachteile:
- Es hat eine komplexe Bedienung,
- Einige Drittprogramme können nur vom Administrator installiert werden,
- Portierungen für Linux-Distributionen stehen für viele Hard- und Software-Entwickler nicht im Fokus,
- der Updateprozess ist mitunter sehr komplex, nicht alle Versionen erhalten Langzeit-Support,
- und einige professionelle Programme funktionieren unter Linux nicht.

Aufgaben eines Servers

Server können viele unterschiedliche Aufgaben übernehmen. Zu den wichtigsten Aufgaben gehören die folgenden:
- Webserver: Speichern und Liefern von Webseiten an Webbrowser,
- Fileserver: Speicherung von Dateien,
- Datenbankserver: Speichern und Verwalten von Datenbanken,
- Gameserver: verwalten von Spieldaten für Multiplayerspiele,
- Proxyserver: Weiterleiten von Anfragen und
- DNS-Server: Namensauflösung von Hostnamen in IP-Adressen.
