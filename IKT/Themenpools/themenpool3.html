A - John von Neumann Architektur

John von Neumann Maschine

Frühe mechanische Rechenmaschinen waren höchst komplex und konnten nur streng sequenziell arbeiten. John von Neuman entwickelte daher die John von Neumann Maschine, dessen Regel, die John von Neuman Architektur, noch bis heute einen großen Einfluss auf unsere Hardware hat. Diese sind folgende:

  1. Ein programmgesteuerter Rechner besteht aus:
    1. Central Processing Unit (CPU) mit
      1. Arithmetical Logical Unit (ALU).
      2. Steuereinheit.
    2. Speicher
    3. Ein- und Ausgabe Einheiten.
    4. Interne Datenwege (Busse).
  2. Die Zahlen werden im Rechner binär dargestellt ( z.B 01010101 ).
  3. Die Struktur ist unabhängig von der Funktion, Aufgaben werden durch Programme gelöst.
  4. Programme und benötigte Daten sind in einem gemeinsamen Speicher. Die Speicherplätze
    sind gleich lang und werden über Adressen einzeln angesprochen ( Register )
  5. Befehle geben nur die Speicheradresse an, wo die Daten abgelegt sind, und nicht die Daten selbst ( Pointer )

EVA-Prinzip

Das EVA-Prinzip ist ein Prinzip der Datenverarbeitung. Es besteht aus drei Schritten:

1. Eingabe.
2. Verarbeitung durch Datenverarbeitungsanlagen.
3.Ausgabe.

Daher kann es auch bei nicht informatischen Fällen angewendet werden, wird jedoch meist zur Verbindung von Hardware angewandt.

Hardware von John von Neumann Maschine

In der Neumann Architektur werden nur wenige Komponente verwendet. Zu diesen gehören:

- Die CPU ( mit ALU und Kontroll/Steuereinheit, aber ohne Zwischenspeicher),
- Ein Speicher,
- Ein- und Ausgabemöglichkeiten,
- Busse ( Mainboard )

while True:
    eingabe = input("Du: ").lower()  # Eingabe vom Benutzer, alles in Kleinbuchstaben

    if eingabe == "exit":
        print("Chatbot: Auf Wiedersehen!")
        break  # Schleife beenden

    # Antwort suchen und ausgeben

        print("Chatbot:", antworten[eingabe])

        print("Chatbot: Das habe ich leider nicht verstanden.")

B - Technicken der Steganographie

Steganographie

Die Steganographie beschreibt das Verstecken von Nachrichten in anderen Nachrichten oder Objekten. Die versteckte Nachricht kann hierbei nicht einfach so gelesen werden, sondern muss zuerst decodiert werden. Das Wort kommt vom griechischen steganos (versteckt) und graphia (schreiben). Im Vergleich zur Kryptographie versteckt Steganographie nicht nur den Inhalt der Nachricht, sondern auch die Nachricht an sich. So ist das Ziel nicht zwingend eine besonders komplexe Verschlüsselung, sondern viel mehr niemanden dazu zu bringen, die Nachricht lesen zu wollen.

Früher wurden z.B. Nachrichten mit Spezialtinte versteckt. Eine weitere Methode war auch das Verstecken von Nachrichten in Morsecode als Knoten auf einem Faden, welcher zu einem Kleidungsstück genäht wurde. Die Spartaner verwendeten sogenannte Skytalen, Stöcke mit einem gewissen Durchmesser um welche ein Lederstreifen mit einer Botschaft gewickelt wurde

Semagramme - Determinative

Ein Semagramm, bzw. Determinativ ist in antiken Sprachen ein stummes Zusatzzeichen, welches die Bedeutung des beistehenden Wortes verändern kann. Weiters können bei Hieroglyphen auch gesamte Wörter als Semagramme bezeichnet werden. Generell sind Semagramme Bilder, welche eine gewisse Bedeutung vermitteln, welche meist nur einer gewissen Gruppe bekannt ist.

Sonagramm

Ein Sonagramm ist ein Schall-Spektrogramm. Sprich eine visuelle Darstellung von Schall. Die x-Achse repräsentiert hierbei die Zeit, während die y-Achse die Frequenz darstellt. Die Amplitude wird mit der Stärke der Einfärbung an einer Stelle gemessen. So kann man in einer Audio-Datei Bilder verstecken. Als Beispiel dient hier der Song ‚My Violent Heart‘, welcher das Albumcover in sich versteckt hat.

Least Significant Bit

Hierbei werden jeweils die zwei niederwertigsten Bits von jeder Farbkomponente ersetzt. Wenn man diese dann extrahiert und normalisiert, erhält man ein Bild, welches zuvor in einem anderen Bild versteckt war. Als Beispiel ein Bild einer Katze, welches im Bild eines Baumes versteckt war.

RSA-System

Das RSA-System ist ein asymmetrisches Verschlüsselungserfahren, welches seit 1977 von Rivest, Shamir und Adleman öffentlich beschrieben wurde. Es benötigt 3 Zahlen von denen 2 öffentlich sind. Diese Zahlen werden folgendermaßen ermittelt: n = p * q, wobei p und q sehr große (>100 Stellen) Primzahlen sind. Die nächste der drei Zahlen ist e, welche teilerfremd zur Zahl (p-1)(q-1) ist. e und n bilden gemeinsam den public Key, d.h. diese beiden können ohne Probleme veröffentlicht werden. Um eine Nachricht zu verschicken, wird die Nachricht m in folgende Formel eingesetzt: c = me (mod n), wobei c die verschlüsselte Nachricht ist. Um sie zu entschlüsseln, wird nun die Zahl d benötigt. Diese wird mit folgender Formel errechnet: de = 1 (mod (p-1)(q-1)). Die Formel zur Entschlüsselung ist nun m = cd (mod n). Hier ist es daher sehr deutlich sichtbar, dass alle zur Verschlüsselung benötigten Faktoren (n, e) ohne Probleme veröffentlicht werden können, da der zur Entschlüsselung benötigte Faktor (d), bzw. die zur Errechnung von jenen benötigten Faktoren (p, q) nahezu unmöglich mit den zur Verschlüsselung benötigten Faktoren (n, e) errechnet werden können.
